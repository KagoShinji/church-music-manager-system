# Contributing to Church Music Manager

First off, thank you for considering contributing to the Church Music Manager! It's people like you that make this a great tool for church communities.

## Code of Conduct

By participating in this project, you agree to abide by the terms of our Code of Conduct. We expect all contributors to maintain a professional and respectful environment.

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issue Tracker.
- Describe the bug in detail and provide steps to reproduce.
- Include information about your browser and OS.

### Suggesting Enhancements
- Open a new issue with the tag `enhancement`.
- Describe why this feature would be useful and how it should work.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue that pull request!

## UI/UX Guidelines

- All new components must adhere to the **sharp-edge philosophy** (no `border-radius`).
- Stick to the **Blue/White color palette**.
- Use **Lucide React** for icons.
- Ensure responsive design.

## Developing Locally

Refer to [docs/SETUP.md](docs/SETUP.md) for environment configuration and local development steps.
